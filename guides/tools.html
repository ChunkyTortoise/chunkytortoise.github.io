What Is MCP?

The Model Context Protocol (MCP) is an open standard that lets AI assistants like Claude connect to external tools and data sources. Instead of copy-pasting data into chat, MCP servers expose structured tools that Claude can call directly — query a database, organize files, search a knowledge base, or check system metrics.

Each MCP server is a lightweight process that registers its tools with the AI client. Claude Desktop, VS Code, and other MCP-compatible clients discover available servers and present their tools as callable functions during conversations.

MCP Toolkit

233 Tests 6 Servers 32 Tools

Six production-ready MCP servers built on FastMCP v2, plus an auto-discovery CLI and a Streamlit playground for testing tools interactively.

SQLite CRUD Server

Full create, read, update, delete operations on SQLite databases. Schema inspection, query execution, and table management through MCP tools.

File Organizer Server

Automated file sorting, renaming, and directory organization. Rule-based and pattern-matching file management.

TF-IDF Knowledge Base Server

Document indexing and semantic search using TF-IDF with scikit-learn. Add documents, search by query, and retrieve ranked results.

System Monitor Server

CPU, memory, disk, and process monitoring. Real-time system health checks accessible through Claude Desktop.

Git Analytics Server

Repository statistics, commit history analysis, contributor metrics, and branch comparisons using GitPython.

Task Tracker Server

Task creation, status tracking, priority management, and project organization. Persistent storage with SQLite backend.

The toolkit also includes an Analytics and Cost Tracking server that logs tool usage, tracks token costs, and generates usage reports across all servers.

# Auto-discover and list all servers

# Launch the Streamlit playground
streamlit run playground.py

MCP Server Toolkit

412 Tests 9 Servers PyPI: mcp-server-toolkit 0.2.0

Nine MCP servers focused on connecting AI workflows to external business tools. Published on PyPI as mcp-server-toolkit==0.2.0. This toolkit was built for the YAML-Driven AI Workflow Engine case study.

Servers connect to calendar, email, Slack, GitHub, and custom business tools. YAML-based pipeline configuration allows non-technical users to define workflows without writing Python. Server-Sent Events (SSE) streaming provides real-time updates during execution.

AgentForge

550 Tests Pip-Installable Multi-Agent Mesh

Async-only LLM orchestrator with a unified interface across Claude, Gemini, OpenAI, and Perplexity. Includes retry/backoff, per-request cost tracking, and structured JSON output.

  • ReAct Agent Loop — Reason-Act-Observe pattern for tool-using agents
  • Multi-Agent Mesh — Agents coordinate through shared memory and message passing
  • Tool Registry — Register tools once, auto-convert to each provider's format
  • Prompt Template Engine — Reusable prompt templates with variable injection
  • Evaluation Framework — Benchmark models against ground-truth datasets
  • Token-Bucket Rate Limiting — Per-provider rate limiting to stay within API quotas
  • Model Registry — Centralized config for model capabilities, pricing, and limits

# Install
pip install agentforge

# Quick start
from agentforge import AgentForge

forge = AgentForge()
response = await forge.chat("claude", "Summarize this document...")
print(response.text, response.cost)

zoxide

Rust Shell Utility

A smarter cd command that learns your habits. zoxide tracks which directories you visit most frequently and lets you jump to them with partial name matches. Type z project instead of cd ~/code/clients/acme/project.

  • Frecency ranking — Combines frequency and recency to rank directory matches
  • Fuzzy matching — z foo bar matches any path containing both "foo" and "bar" in order
  • Interactive selection — zi opens an fzf-powered picker when multiple directories match
  • Shell integration — Works with bash, zsh, fish, Nushell, PowerShell, and others
  • Drop-in replacement — Alias cd to z and your existing muscle memory keeps working

# Install (macOS)
brew install zoxide

# Install (Linux)
curl -sSfL https://raw.githubusercontent.com/ajeetdsouza/zoxide/main/install.sh | sh

# Add to ~/.bashrc or ~/.zshrc
eval "$(zoxide init bash)"  # or zsh, fish, etc.

# Usage
z myproject      # jump to best match for "myproject"
zi myproject     # interactive selection with fzf
z foo bar        # match path containing both "foo" and "bar"

I use zoxide across every development machine. Once your shell learns your project directories, navigation becomes nearly instant — no more tabbing through deep folder hierarchies.
